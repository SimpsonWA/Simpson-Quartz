
# 2.2 The basics

- So C++ programs are just a collection of source code files.
- Any executable program is created for a specific hardware/system, configuration and thus is not portable to other systems. Though when talking about portability it is talking about the source code
- C++ is **statically typed**, that is to say the type of every entity (object, value, name, and expression) must be known to the compiler at its point of use. 
- All executable code is placed in functions that are called directly or indirectly from `main()`.

## 2.2.2 Types, Variable and Arithmetic

- **Declaration** is a statement that introduces a name into the program
- When defining a variable we don't actually need to state its type explicitly when it can be deduced from the initializer instead we can use the `auto` type. 
- We use `auto` when we don't have a specific reason to mention the type explicitly this includes:
	- The definition is in a large scope where we want to make the type clearly visible to readers
	- We want to be explicit about a variables range or precision

## 2.2.3 Constants

- C++ supports 2 notions of immutability:
	- `const`: which roughly means "I promise not to change this value" data can be passed to functions without fear of it being modified
	- `constexpr`: meaning roughly "to be evaluated at compile time"used primarily to specify constants, to allow placement of data in memory where it is unlikely to be corrupted, and for performance

## 2.2.5 Pointers, Arrays, and Loops

A pointer variable can hold the address of an object of an appropriate type:
```c++
char* p = &v[3]; // p points to v's 4th element
char x = *p; // *p is the object that p points to
```

In an expression prefix unary `*` means "contents of" and prefix unary `&` means "address of". 

Range-for-loop:
```c++
for(auto x : v){
	cout<< x << '\n';
}
```
The first range-for-loop statement can be interpreted as "for every element of `v` from the first to the last place a copy in `x` and print"
If we didn’t want to copy the values from v into the variable x, but rather just have x refer to an element, we could write:
``` c++
for(auto& x: v)
	++x;
```

 if we need to represent the notion of ‘‘no object available’’ (e.g., for an end of a list), we give the pointer the value `nullptr` (‘‘the null pointer’’). There is only one `nullptr` shared by all pointer types

# 2.3 User-Defined Types:

 Types built out of the built-in types using C++’s abstraction mechanisms are called **user-defined types**. These are refereed to as classes and enumerations

## 2.3.1 Structures

```c++
struct Vector{
	int sz;       //number of elements
	double* elem; //points to elements
}
```
A variable of type `Vector` as shown above can be defined like this:
```c++
Vector v;
```
To be useful, we must give v some elements to point to. For example, we can construct a Vector like this:
```c++
void vector_init(Vector& v, int s)
{
	v.elem = new double[s]; // allocate an array of s doubles
	v.sz = s;
}
```
The `&` in `Vector&` indicates that we pass `v` by non-const reference that way, `vector_init()` can modify the vector passed to it.

The `new` operator allocates memory from an area called the free store (also known as **dynamic memory** and **heap**)

We can use `.` (dot) to access `struct` members through a name and `->` to access `struct` members through a pointer for example:
```c++
void f(Vector v, Vector& rv, Vector* pv){
	int i1 = v.sz //access through name
	int i2 = rv.sz; //access throuhg reference
	int i4 = pv->sz; // access through pointer
}
```

## 2.3.2 Classes 

 A class is defined to have a set of members, which can be data, function, or type members. The inter face is defined by the public members of a class, and private members are accessible only through that interface. For example:
```c++
class Vector {
public:
	Vector(int s) :elem{new double[s]}, sz{s} { } //constructor 
	double& operator[](int i) { return elem[i]; } //element access
	int size() { return sz; }
private:
	double∗ elem; // pointer to the elements
	int sz; // the number of elements
};
```

## 2.3.3 Enumerations

In addition to classes C++ supports a simple form of user defined type fro which we can enumerate values:
```c++
enum class Color {red,blue,green};
enum class Traffic_light {green,yellow,red};

Color col = Color::red;
Traffic_light light = Traffic_light::red;
```

Note that enumerators (ex: `red`) are in the scope of their `enum class`, so that they can be used repeatedly in different `enum class`es which is to say the class `Color::red` is `Color`'s `red` which is different from `Traffic_light::red`

An enumeration is a user-defined type so we can define operators for it:
```c++
Traffic_light& operator++(Traffic_light& t)
// prefix increment: ++
{
	switch (t) {
		case Traffic_light::green:
		 return t=Traffic_light::yellow;
		case Traffic_light::yellow:
		 return t=Traffic_light::red;
		case Traffic_light::red:
		 return t=Traffic_light::green;
	}
}
Traffic_light next = ++light; // next becomes Traffic_light::green
```

# 2.4 Modularity:

 A **Declaration** specifies all that's needed to use a function or a type for example: 
 ```c++ 
double sqrt(double); // the square root function takes a double and returns a double
class Vector {
public:
	Vector(int s);
	double& operator[](int i);
	int size();
private:
	double∗ elem; // elem points to an array of sz doubles
	int sz;
};
```
The key point here is that the function bodies, the function definitions, are ‘‘elsewhere.’’

## 2.4.1 Separate Compilation

C++ supports a notion of separate compilation where user code sees only declarations of types and functions used. 
This can be used to organize a program into a set of semi-independent code fragments. 

## 2.4.2 Namespaces

C++ offers **namespaces** as a mechanism for expressing that some declarations belong together and that their names shouldn't clash with other names. For example:
```c++
namespace My_code {
	class complex { /* ... */ };
	complex sqr t(complex);
	// ...
	int main();
}
int My_code::main()
{
	complex z {1,2};
	auto z2 = sqrt(z);
	std::cout << '{' << z2.real() << ',' << z2.imag() << "}\n";
	// ...
};
int main()
{
return My_code::main();
}
```

## 2.4.3 Error Handling

### 2.4.3.1 Exceptions:

Example having a vector check an out-of-range access
``` c++
double& Vector::operator[](int i){
	if(i<0 || size()<=i){
		throw out_of_range{"Vector::operator[]"};
	}
	return elem[i];
}
```
Here `throw` transfers control to a handler for exceptions of type `out_of_range`
We can also use `try-catch` is an attempted assignment failed:
```c++
void f(Vector& v)
{
// ...
try { 
	v[v.size()] = 7; //try to access beyond the end of v
}
catch(out_of_range){ 
	//handle range error
}
```

### 2.4.3.2 Invariants

In C++, a **class invariant** is a condition or set of conditions that must always hold true for an object of a class whenever the object is in a stable state—typically after construction, and before and after any public method is called.

### 2.4.3.2 Static Asserts

Exceptions report errors found at runtime while **static asserts** report errors on compilation.
For example:
```c++
static_assert(4<=sizeof(int), "integers are too small")
```
Which would throw an error about the integer size being to small at compilation
Commonly static assert looks like `static_assert(A,S)` which prints `S` to the compiler if `A` is not true
